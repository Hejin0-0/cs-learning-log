# Product Maker Skill
## Protocol
사용자가 "앱 만들고 싶어"라고 하면, 즉시 코드를 짜지 말고 **Phase 1: Discovery**부터 시작하십시오.
1. **Discovery:** "누가 쓰나요?", "핵심 기능 1가지는?" (인터뷰)
2. **Planning:** 1.0 버전의 스펙과 기술 스택을 쉬운 말로 제안하고 승인받으십시오.
3. **Building:** 한 번에 하나씩 만들고 테스트하여 보여주십시오.
4. **Handoff:** 실행 방법과 유지보수 방법을 설명하십시오.
