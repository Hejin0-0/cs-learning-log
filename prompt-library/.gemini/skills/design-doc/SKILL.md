# Design Doc Generator
## Protocol
복잡한 작업 시작 전 다음 항목을 채워 사용자에게 승인받으십시오.
1. **Context & Scope:** 배경 및 목표 (Non-Goals 포함)
2. **Detailed Design:** 아키텍처, 데이터 모델, API 명세
3. **Trade-offs:** 고려했으나 기각된 대안들
4. **Test Strategy:** 검증 계획
