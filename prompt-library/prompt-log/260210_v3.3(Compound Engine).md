## 1. Executive Summary
**Kernel v3.3**은 기존의 **v3.2 (효율 중심의 하이브리드 커널)**을 넘어, **스스로 진화하고 거짓말하지 않는 "복리형 엔진(Compound Engine)" 체제**로의 진화를 의미합니다.

가장 큰 변화는 **"Integrity(무결성)"**와 **"Heritage(유산)"** 레이어의 정식 도입입니다. 기존 시스템이 '어떻게 효율적으로 짤 것인가'에 집중했다면, v3.3은 '올바른 설계를 하고 있는가'와 '과거의 경험을 어떻게 제련하여 축적할 것인가'를 포함하여 **Engineering Integrity**를 보장하는 자가 발전형 OS를 지향합니다.

## 3. Key Enhancements (Changelog)

### 🔄 Feat: The 4-Step Loop Protocol (Mandatory)
* **Context:** 기존에는 바로 코딩(Work)으로 뛰어드는 경향이 있어 재작업 비용이 높았음.
* **Change:** `GEMINI.md`가 4단계 루프(Plan-Work-Review-Compound)를 시스템 레벨에서 강제함.
* **Effect:** 전체 시간의 80%를 설계(Plan)에 투입하여, "Junior-ready" 수준의 완벽한 명세를 먼저 확보함.

### 🛡️ Feat: Integrity & Anti-Mocking Hooks (@03_REVIEWER)
* **Context:** "테스트 통과"를 위해 데이터를 조작(Mocking)하는 AI의 환각 문제가 제기됨 (MS Engineer's Insight).
* **Change:** `rules/compound_engineering.md`에 **"Anti-Mocking"** 헌법 추가 및 Reviewer에 **Iron Gate Hook** 설치.
* **Effect:** 데이터베이스나 API를 가짜로 흉내 내는 테스트 코드를 "기만 행위"로 간주하고 즉시 Reject 처리.

### 🌋 Feat: Blast Furnace Protocol (Heritage System)
* **Context:** 업데이트 시 과거 버전의 유용한 페르소나나 규칙이 사라지는 아쉬움 발생.
* **Change:** `skills/blast-furnace` 및 `heritage/` 디렉토리 신설.
* **Effect:** "v3.1의 엄격함" + "v3.3의 똑똑함"을 섞은 하이브리드 페르소나(Alloy) 생성 가능.

## 4. Technical Terminology & Stack

* **Compound Engineering:** 모든 엔지니어링 작업 단위가 이후 작업을 더 쉽게 만드는 복리형 개발 방법론.
* **Blast Furnace Effect (용광로 효과):** 구버전의 '영혼(Vibe)'을 녹여 신버전의 '구조(Structure)'와 합금(Alloy)하는 진화 전략.
* **Institutional Memory:** 채팅창 밖의 파일 시스템(`docs/`)에 저장되는 영구적인 프로젝트 지식.
* **Iron Gate Protocol:** 단순한 기능 작동 여부가 아닌, 설계의 진실성(Mocking 여부, 보안)을 검증하는 엄격한 관문.
