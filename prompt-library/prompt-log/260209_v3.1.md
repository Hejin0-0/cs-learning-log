## 1. Executive Summary
**Kernel v3.1**은 기존의 **v3.0 (기능 중심의 전문가 집단)**을 넘어, **완벽한 E2E(End-to-End) 제품 개발이 가능한 "12인 스쿼드(Squad)" 체제**로의 확장을 의미합니다.

가장 큰 변화는 **"Why(기획)"**와 **"Data(데이터)"** 레이어의 정식 도입입니다. 기존 시스템이 '어떻게 구현할 것인가(How)'에 집중했다면, v3.1은 '무엇을 왜 만드는가'와 '데이터를 어떻게 보존할 것인가'를 포함하여 **Start-up to Enterprise** 레벨의 풀 사이클을 커버합니다.

## 3. Key Enhancements (Changelog)

### 🚀 Feat: Dynamic Leadership Injection (@00_PO)
* **Context:** 기존에는 개발자 관점의 설계만 존재하여 "사용자에게 감동을 주는" 요소가 부족했음.
* **Change:** 두 가지 상반된 리더십 페르소나를 도입하여 **Session Context Injection** 시점에 선택 가능하도록 함.
    * `Type A (Purist)`: Steve Jobs/Jony Ive (미니멀리즘, 완벽주의)
    * `Type B (Curator)`: Miyamoto/Wintour (재미, 트렌드, 편집)

### 💾 Feat: Schema-First Development (@11_DBA)
* **Context:** 프론트엔드 중심 개발 시 DB 정규화가 깨지거나 N+1 문제가 빈번히 발생함.
* **Change:** 코드 작성 전 **Schema Artifact** 승인을 강제하는 **Pre-commit Gate** 도입.
* **Effect:** 데이터 무결성(ACID) 보장 및 쿼리 최적화가 설계 단계에서 확정됨.

### ⚡ Refactor: Context Window Optimization (Memory Efficiency)
* **Problem:** 12명의 전문가를 모두 로딩하면 LLM의 **Context Window(토큰 제한)** 를 과도하게 점유할 우려가 있음.
* **Solution:** **"Lazy Loading & Role Separation"** 패턴 적용.
    * **Rotation System:** 세션 시작 시 `@00`과 `@00-sub` 중 하나만 활성화하여 컨텍스트 오염 방지.
    * **Artifact Protocol:** 모든 Role이 "잡담(Fluff)"을 제거하고 구조화된 결과물(Artifact)만 출력하므로, 대화 히스토리의 토큰 효율이 **v3.0 대비 약 40% 개선**됨.

## 4. Technical Terminology & Stack

* **SoC (Separation of Concerns):** 기획(PO), 설계(Arch), 데이터(DBA), 구현(Builder)의 역할이 코드 레벨에서 완벽히 분리됨.
* **Water-Scrum-Fall:** 기획과 설계는 철저하게(Waterfall), 구현은 민첩하게(Scrum), 검증은 엄격하게 진행하는 하이브리드 방법론 채택.
* **Single Source of Truth (SSOT):** `GEMINI.md`가 유일한 시스템 커널로서 모든 Role의 라우팅 테이블 역할을 수행.
* **Zero Ambiguity Protocol:** 자연어가 아닌 **Artifact(JSON/Markdown)** 기반 통신으로 환각(Hallucination) 최소화.
