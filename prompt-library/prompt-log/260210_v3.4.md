# Release Note: Kernel v3.4 - The Agentic Orchestrator
**Date:** 2026-02-10
**Tag:** `v3.4-agentic-future`
**Previous:** `v3.3-compound-integrity`
**Ref:** Anthropic 2026 Trends Report

---

## 1. Executive Summary
**Kernel v3.4**은 기존의 강력한 **4-Step Loop** 구조 위에 Anthropic 2026 트렌드(Orchestration, Security, Persistence)를 통합한 **"자율 지휘형 엔진"**입니다.
사용자는 익숙한 프로세스(Plan-Work-Review-Compound)를 그대로 사용하되, 그 내부에서는 **Red Team 보안 점검**, **장기 기억 저장**, **전문가 스쿼드 위임**이 자동으로 수행됩니다.

---

## 2. Key Features (The 2026 Shift)

### 🛡️ Security-First Architecture (Trend 8)
* **New Rule:** `rules/security_first.md`
* **Impact:** Plan 단계에서 보안 위협을 식별하고, Review 단계에서 **"Red Team"** 모드로 공격 시뮬레이션을 수행하여 방어력을 내재화함.

### ⏳ Long-Running Persistence (Trend 3)
* **New Artifact:** `docs/progress/status.md`
* **Impact:** **Compound 단계**가 단순히 지식만 쌓는 것이 아니라, 프로젝트의 **진행 상태(State)**를 기록하여 대화가 끊겨도 연속성을 보장함.

### 🎹 Orchestration over Implementation (Trend 1, 2)
* **Change:** Work 단계가 "직접 코딩"에서 **"스쿼드 위임(Delegation)"**으로 진화.
* **Impact:** `@04_DOMAIN_EXPERT` 등 비기술적 이해관계자를 위한 통역 기능 추가 및 전문 에이전트 협업 강화.

---

## 3. Migration Guide
1.  **Install:** `rules/security_first.md`, `rules/agentic_engineering.md` 생성.
2.  **Update:** `GEMINI.md`를 v4.0으로 업데이트 (4-Step Loop 구조 유지).
3.  **Init:** `mkdir -p docs/progress` 실행.

> **Maintainer's Note:**
> "가장 좋은 혁신은 익숙한 것 안에서 일어나는 변화입니다. 4-Step Loop는 그대로지만, 그 엔진은 2026년형으로 교체되었습니다."
